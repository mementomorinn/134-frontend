Выбор редактора кода

Первым шагом в создании нашего сайта является выбор и настройка редактора кода. На данный момент их существует огромное множество и каждый имеет свои особенности и преимущества. Вот некоторые из самых популярных:
  • 1. Visual Studio Code (VS Code)
    Многофункциональный редактор от Microsoft, поддерживающий расширения для различных языков и фреймворков. Имеет встроенную поддержку Git, терминал и отладку.
  • 2. Sublime Text
    Легкий и быстрый редактор с элегантным интерфейсом. Поддерживает плагины, но некоторые функции могут требовать дополнительной настройки.
  • 2. Atom
    Редактор от GitHub, который можно легко настраивать и расширять. Он поддерживает плагины и интеграцию с Git.
  • 2. WebStorm
    Коммерческий редактор от JetBrains, специально разработанный для JavaScript и веб-разработки. Обладает мощными инструментами для работы с фреймворками и целостной интеграцией.
В данном курсе не будет сильного углубления в программирование, поэтому нам подойдет практически любой из существующих редакторов кода. Более того, наш код можно писать даже в обычном блокноте или любом другом текстовом редакторе!
Но все же я рекомендую вам начать работу в Visual Studio Code, так как он предлагает богатую экосистему расширений и активное сообщество, что делает его хорошим инструментом для начинающих веб-разработчиков. Скачать его можно по данной ссылке.

Установка плагинов

Итак, после успешной установки Visual Studio Code открываем его и заходим во вкладку Extensions (магазин расширений). С помощью поиска находим и устанавливаем следующие расширения:
  • 1. Auto Rename Tag
    При редактировании открывающего тега автоматически меняет и закрывающий. То же самое, с точностью до наоборот, происходит и в том случае, если меняют закрывающий тег.
  • 2. Live Server
    Позволит запускать код в браузере и обновлять изменения после нажатия комбинации клавиш ctrl+S.
  • 3. VS HTML to CSS
    Позволит скопировать из выделенного фрагмента HTML разметки только классы (alt + X) и вставить их в CSS.
  • 4. Dracula Official
    Просто красивая тема, не обязательна к установке. Тема меняется в настройках (File -> Preferences -> Color Theme).
  • 5. Material Icon Theme
    Также не обязательна к установке. Добвляет различные иконки для файлов и каталогов проекта.

Файлы проекта

Для верстки сайтов необходимо создать структуру файлов и каталогов, которая обеспечит удобное управление проектом. Основные файлы и каталоги должны включать:
  • 1. index.html
    Главный HTML-файл, который загружается первым. Он содержит основную разметку сайта.
  • 2. css/
    Каталог для хранения файлов стилей. Внутри этого каталога обычно создается файл styles.css , где находятся все CSS-правила.
  • 3. images/
    Каталог для хранения изображений и графики, используемых на сайте.
  • 4. fonts/
    Каталог для шрифтов, если используются пользовательские шрифты.
Пример структуры каталога:
my-first-project
├──index.html
├──css/
│ └── styles.css
├── images/
└── fonts/
В более сложных проектах можно встретить и другие файлы, например, README.md (файл с информацией о проекте, его структуре и особенностях), js (каталог для JavaScript-файлов, которые содержат функциональность сайта) и assets (дополнительный каталог для хранения медиафайлов, иконок и других ресурсов).
Создание такой структуры поможет организовать код и упростит процесс разработки и поддержки сайта.
